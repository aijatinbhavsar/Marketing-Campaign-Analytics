# 📖 Data Dictionary – ShopEasy Marketing Analytics

Defines every column used across the 5 source tables, after cleaning. All definitions reflect the **post-CTE cleaned layer** used in Power BI — not the raw source values.

---

## 🧑 Customers

| Column | Type | Description | Notes |
|---|---|---|---|
| `CustomerID` | INT | Unique customer identifier | PK — NULLs excluded in CTE |
| `CustomerName` | VARCHAR | Full name of the customer | Trimmed of whitespace |
| `Email` | VARCHAR | Customer email address | Lowercased + trimmed |
| `Gender` | VARCHAR | Customer gender: `Male` / `Female` | Corrected via first-name pattern matching (source had mismatches) |
| `Age` | INT | Customer age | NULLed if < 10 or > 95 (out-of-range values) |
| `Country` | VARCHAR | Country from Geography table | Joined via `GeographyID` in CTE |
| `City` | VARCHAR | City from Geography table | Joined via `GeographyID` in CTE |

---

## 🛍️ Products

| Column | Type | Description | Notes |
|---|---|---|---|
| `ProductID` | INT | Unique product identifier | PK — NULLs excluded |
| `ProductName` | VARCHAR | Display name of the product | Trimmed of whitespace |
| `Category` | VARCHAR | Product category | Uppercased + trimmed |
| `Price` | DECIMAL | Unit price of the product | Negative values converted via `ABS()` — used in `Total Revenue` DAX measure via `RELATED()` |
| `PriceTier` | VARCHAR | Budget / Mid-Range / Premium | **Derived column** — not in source. Logic: `< 50` → Budget, `50–200` → Mid-Range, `> 200` → Premium. Added per Jane Doe's request for price-tier performance analysis |

---

## 🗺️ Customer_Journey

The core fact table — records every user interaction with the platform.

| Column | Type | Description | Notes |
|---|---|---|---|
| `JourneyID` | INT | Unique event identifier | PK — NULLs excluded |
| `CustomerID` | INT | FK → Customers | — |
| `ProductID` | INT | FK → Products | — |
| `VisitDate` | DATE | Date of the interaction | Cast from DATETIME; future dates excluded |
| `Stage` | VARCHAR | Funnel stage: `Home Page` / `Product Page` / `CHECKOUT` | Standardized casing (e.g. `PRODUCTPAGE` → `Product Page`) |
| `Action` | VARCHAR | User action: `VIEW` / `CLICK` / `DROP-OFF` / `PURCHASE` | Uppercased + trimmed |
| `Duration` | FLOAT | Time spent on page (seconds) | NULLs imputed with average duration per Stage using window function |

**Duplicate rule:** Rows with identical `(CustomerID, ProductID, VisitDate, Stage, Action)` are deduplicated — first `JourneyID` wins.

## ⭐ Customer_Reviews

| Column | Type | Description | Notes |
|---|---|---|---|
| `ReviewID` | INT | Unique review identifier | PK — NULLs excluded; used as join key with `full_score_reviews` |
| `CustomerID` | INT | FK → Customers | — |
| `ProductID` | INT | FK → Products | — |
| `ReviewDate` | DATE | Date review was submitted | Future dates excluded (`<= GETDATE()`) |
| `Rating` | INT | Star rating 1–5 | Clamped: values < 1 set to 1, values > 5 set to 5 |
| `ReviewText` | VARCHAR | Free-text review content | Trimmed; multiple spaces collapsed |

**Duplicate rule:** Rows with identical `(CustomerID, ProductID, ReviewDate, ReviewText)` are deduplicated.

## 📣 Engagement_Data

Records campaign-level content performance metrics.

| Column | Type | Description | Notes |
|---|---|---|---|
| `EngagementID` | INT | Unique engagement event identifier | PK |
| `ProductID` | INT | FK → Products | — |
| `ContentType` | VARCHAR | Type of content: `VIDEO` / `BLOG` / `SOCIAL` / `OTHER` | NULL → `OTHER` via COALESCE |
| `EngagementDate` | DATE | Date of engagement event | Cast from DATETIME |
| `Likes` | INT | Number of likes on the content | NULL → 0; negative values made absolute via ABS() |
| `Views` | INT | Number of content views | Split from `ViewsClicksCombined` string column |
| `Clicks` | INT | Number of content clicks | Split from `ViewsClicksCombined` string column |
| `CTR` | FLOAT | Click-Through Rate = (Clicks / Views) × 100 | Calculated in CTE; 0 when Views = 0 to avoid division error |

**Original issue:** `Views` and `Clicks` were stored as a single string `"1250-47"` in `ViewsClicksCombined`. Split using `CHARINDEX` + `LEFT` + `SUBSTRING`. NULL values coalesced to `'0-0'` before splitting.

**Duplicate rule:** Same `(ContentType, EngagementDate)` pair keeps the row with highest `Likes`.

## 🤖 full_score_reviews (Python Output)

Generated by `03_sentiment_analysis/sentiment_analysis.py` — not a source table.

| Column | Type | Description | Notes |
|---|---|---|---|
| `ReviewID` | INT | FK → Customer_Reviews | Join key back to the reviews table |
| `CustomerID` | INT | FK → Customers | Carried through from source query |
| `Customer_Feedback_Score` | INT | Sentiment score 1–10 | Derived from VADER compound score: `((compound + 1) / 2) × 9 + 1` |

**Score interpretation:**
- 1–3 → Negative sentiment
- 4–6 → Neutral
- 7–10 → Positive sentiment

| `Day` | INT | Day number |
| `Day Name` | VARCHAR | Monday / Tuesday … |
| `Day of Week Number` | INT | 1–7 |
| `Month Number` | INT | 1–12 |
| `Month Name` | VARCHAR | January … December |
| `Month Short` | VARCHAR | Jan … Dec |
| `Quarter` | VARCHAR | Q1 / Q2 / Q3 / Q4 |
| `Year` | INT | Calendar year |
| `Year Month` | VARCHAR | e.g. 2024-Jan |
| `Year Quarter` | VARCHAR | e.g. 2024-Q1 |
| `Fiscal Year` | VARCHAR | Fiscal year label |
| `Is Weekend` | BOOLEAN | TRUE if Saturday or Sunday |
| `Total` | — | Placeholder for total row label |
